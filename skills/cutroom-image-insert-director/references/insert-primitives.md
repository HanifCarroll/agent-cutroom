# Insert Primitives

Use the smallest insert that solves the beat.

| Primitive | Use When | Notes |
| --- | --- | --- |
| `proof-callout` | The speaker names a result, file, test, metric, or artifact. | Prefer real screenshot or cropped artifact. |
| `screen-insert` | The spoken idea depends on software or a workflow. | Use readable crop; avoid tiny UI. |
| `quote-card` | A short line is the point. | Keep under 10 words if captions remain visible. |
| `diagram` | Relationships matter more than realism. | Use 2-5 nodes; animate reveal if needed. |
| `infographic` | A comparison or sequence needs structure. | Use one chart/shape idea per insert. |
| `generated-scene` | The idea needs an illustrative image and no real asset exists. | Avoid fake product proof. |
| `title-card` | A section reset or chapter break is needed. | Short text only. |
| `lower-third` | Identity or context must be established. | Use once unless the speaker changes. |

If two primitives seem useful, pick the one that removes the most confusion.
